Notes

The tile server can be changed to anything that uses 265x256 sized tiles, the Web Mercator projection, and follows the same {x},{y},{z} convention.

All fetched tiles will be stored permanently in the indexedDB, so they can be loaded quickly and used offline if needed.

Keep in mind that browser-cached data is tied to a specific site URL, so if your robot's IP address changes when transitioning from a local network to offline AP mode, the tiles won't be accessible. One way to get tiles loaded is to first load up the site offline, switch to mobile data to fetch the tiles without refreshing, and then disable it again so rosbridge can reconnect.
